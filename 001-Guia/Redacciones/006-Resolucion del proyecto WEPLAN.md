-El proyecto que hemos hecho:
	El proyecto se llama WePlan 
-De qué se trata:
	Se trata de conectar personas que tengan varias aficiones o actividades ya sean comunes, diferentes o unicas y que puedan comunicarse tanto por mensajes a traves de la aplicaciones o traves de eventos para desconectarse.

-Por qué lo hemos hecho:
Hemos decidido hacer este proyecto porque es algo que estamos buscando para utilizar y no lo encontramos como queremos. Estamos de acuerdo en que hay aplicaciones similares a la que estamos creando, pero les faltan algunos cosas claves que vamos a añadir para mejorar el proyecto y hacerlo más útil. Por ejemplo, en las otras aplicaciones solo las grandes empresas pueden crear actividades, pero en la nuestra todo el mundo puede añadir actividades, aunque no sean gran cosa, porque todos hemos llegado alguna vez a un punto en el que nos sentimos perdidos y necesitamos compañía para tomar un café y charlar sobre cosas sin importancia para sentirnos mejor.

-Con quién lo hemos hecho:
	Se ha creado por Ayman Mouzakki y Bryan Glot Fong

-Cómo nos hemos repartido tareas, quién ha hecho qué:
	Entre los dos, hemos hecho una division de tareas fluida para no sobrecarganos al crear el front-end y el back-end

-Como hemos trabajado en equipo:
	En el front-end, haremos que los usuarios y invitados puedan ver y interactuar entre las 4 paginas fundamentales (Inicio,Notificationes,Actividades y Cuenta). Y en el back-end, donde al gestionar los datos, se controla las acciones hechas por el admin y también control de que pueden ver con las vistas.

-Indicaciones de que conocemos cómo combinar las tecnologías implicadas:
       Front-end:
- HTML para estructurar las 4 páginas principales
- CSS para el diseño y adaptación a dispositivos moviles y ordenadores

	Back-end:
- PHP como núcleo principal para  la gestión de usuarios (registro, login, sesiones), creación y visualización de actividades, sistema de mensajes entre usuarios, control de permisos para los usuarios normales y los administradores.
	Base de datos:
- MySQL para almacenar usuarios, actividades, aficiones, relaciones y vistas

	Flujo de integración:
	 El usuario interactúa con interfaces HTML  estilizada con CSS → PHP procesa las solicitudes mediantes login+sesiones → Consulta/Lee la base de datos → PHP devuelve resultados al front-end al estar conectado con la base de datos.
	
	 Esta flujo de integración permite crear una aplicación completa donde cada tecnología cumple su función óptima: PHP para la web dinámica y lógica compleja y una base de datos SQL para la persistencia de información.

El resultado del proyecto intermodular, WePlan, es una aplicación que resuelva una necesidad humana: la de conexión offline como online de manera simple y eficiente.
